# PensiaMng — מפרט אפיון מלא
## מערכת מקצועית לתכנון וניהול פנסיה בישראל

> **גרסה:** 1.0 | **תאריך:** 2026-07-11
> **הערה חשובה:** כל הערכים הרגולטוריים (תקרות, אחוזים, גילאים, מקדמים) נכונים למועד הכתיבה ומסומנים כפרמטרים ניתנים לעדכון. המערכת חייבת לנהל אותם כטבלת קונפיגורציה עם תאריכי תחולה — לא כקבועים בקוד.

## 1. חזון ומטרות

### 1.1 חזון
מערכת המאפשרת לאדם פרטי או ליועץ פנסיוני להזין את כל הנכסים הפנסיוניים של לקוח, ולקבל תמונה מלאה, מנותחת ומוסברת של מצבו הפנסיוני — כולל תחזיות, תרחישי "מה אם", ניתוח עלויות, כיסויים ביטוחיים והשלכות מס — בשפה ברורה ועם הסברים לכל מספר.

### 1.2 מטרות עיקריות
1. **ריכוז** — כל המוצרים הפנסיוניים במקום אחד (פנסיה, ביטוח מנהלים, גמל, השתלמות, ותיקות, תקציבית).
2. **תחזית** — הקרנת צבירה וקצבה צפויה לגיל פרישה תחת הנחות ניתנות לשינוי.
3. **תרחישים** — סימולציית "מה קורה אם": מוות, אובדן כושר עבודה, עזיבת עבודה, פרישה מוקדמת, שינוי שכר, שינוי מסלול, משיכה.
4. **עלויות** — כימות ההשפעה המצטברת של דמי ניהול והשוואה לממוצעי שוק.
5. **מיסוי** — חישוב הטבות מס בהפקדה, מס במשיכה, פטור על קצבה מזכה וקיבוע זכויות.
6. **הסברה** — כל מסך וכל תוצאה מלווים בהסבר מובנה ("למה זה חשוב", "איך חושב", "מה אפשר לעשות").

### 1.3 מחוץ לתחולה (Out of Scope) — שלב 1
- ייעוץ פנסיוני אישי בפועל (המערכת היא כלי תכנון והמחשה, לא תחליף לבעל רישיון).
- ביצוע פעולות מול הגופים המוסדיים (ניוד, שינוי מסלול) — שלב עתידי.
- ביטוחי בריאות/סיעוד/חיים פרטיים שאינם חלק מהמעטפת הפנסיונית — שלב עתידי.

### 3.3 סוגי מוצרים והשדות הייחודיים

#### א. קרן פנסיה מקיפה (חדשה)
- **תקרת הפקדה חודשית:** 20.5% מפעמיים השכר הממוצע במשק (פרמטר). עודף מנותב לקרן משלימה.
- **מנגנון הבטחת תשואה:** 30% מהנכסים (החליף אג"ח מיועדות, 10/2022) — פרמטר.
- **מסלול ביטוח:** נכות % / שארים % (ברירת מחדל 75%/100%; מסלולים: עתיר חיסכון, עתיר נכות, עתיר שארים וכו').
- **ויתור על כיסוי שארים** (לרווק/ה): דגל + תאריך חידוש (תוקף שנתיים, מתחדש בהצהרה).
- **תקופת אכשרה:** 60 חודשים למצב רפואי קודם — תאריך תחילת כיסוי מלא.
- **גיל תום ביטוח נכות:** 60/62/64/67 (בחירת העמית).
- **פרנצ'יזה / תקופת המתנה לנכות:** פרמטר מסלול.
- **עלות כיסויים חודשית** (נגבית מההפקדה) — מוזנת או מחושבת מטבלת תעריפים.

#### ב. קרן פנסיה כללית (משלימה)
- ללא הבטחת תשואה, ללא תקרת הפקדה; כיסויים ביטוחיים אופציונליים.

#### ג. ביטוח מנהלים (פוליסת ביטוח)
- **דור הפוליסה** (קריטי!): עדיף / משתתפת ברווחים עד 1990 / 1991–2003 / 2004–2012 / מ-2013 (ללא מקדם מובטח).
- **מקדם קצבה מובטח:** קיים/לא + ערכו + גיל הפעלה.
- **סכום ביטוח למקרה מוות (ריסק):** סכום + פרמיה חודשית.
- **כיסוי אובדן כושר עבודה (אכ"ע):** % פיצוי, הגדרה עיסוקית/כל עיסוק, פרנצ'יזה, שחרור מפרמיה.
- **מבנה דמי ניהול היסטורי** (פוליסות ישנות: פרמיה/צבירה שונות מהותית).

#### ד. קופת גמל
- הונית (הפקדות עד 2008 עם זכויות הוניות) / קצבתית; מרכיב הוני ותאריכי נזילות.
- **קופת גמל להשקעה:** תקרת הפקדה שנתית (פרמטר, ~81,700 ₪ ב-2025), נזילה תמיד (מס רווח הון), פטור מלא אם נמשכת כקצבה אחרי 60.
- **IRA (גמל בניהול אישי):** דגל + מגבלות.

#### ה. קרן השתלמות
- תקרת הפקדה מוטבת (שכיר: 10% עד שכר 15,712 ₪; עצמאי: 4.5% עד תקרה שנתית — פרמטרים).
- ותק לנזילות: 6 שנים (3 לגיל פרישה) — תאריך נזילות מחושב.
- אינה מוצר קצבתי אך חלק מהתמונה ההונית לפרישה.

#### ו. קרן פנסיה ותיקה (בהסדר / שלא בהסדר)
- שיטת זכויות (לא צבירה): % זכויות שנצברו, שכר קובע, שיטת ממוצעים / 3 שנים אחרונות.
- דמי ניהול אחידים; גירעון אקטוארי; מגבלות צבירת זכויות (עד 70%).

#### ז. פנסיה תקציבית
- מעסיק (מדינה/רשות), % זכויות (2% לשנה עד 70%), שכר קובע.

#### ח. קצבאות ומקורות נוספים
- קצבת אזרח ותיק (ביטוח לאומי): חישוב אוטומטי לפי ותק + תוספת בן זוג (פרמטרים).
- קצבאות קיימות בתשלום, נכסים נוספים (שכ"ד, תיק השקעות) — להשלמת תמונת הכנסה בפרישה.

### 3.4 היסטוריית תעסוקה (Employment)
לכל תקופת עבודה: מעסיק, תאריכים, שכר מבוטח, אחוזי הפקדה (עובד/מעסיק/פיצויים), סעיף 14, סטטוס פיצויים בעזיבה (נמשכו / רצף קצבה / רצף זכויות) — נדרש לחישובי מס פרישה וטופס 161.

### 3.5 מסלולי השקעה — קטגוריות (לפי רפורמת המסלולים 2023)
מסלולי ברירת מחדל תלויי גיל (מודל חכ"מ/צ'יליאני: עד 50, 50–60, 60+), מניות, עוקב מדד S&P 500, אג"ח, שקלי, הלכה, שריעה, משולב סחיר, קיימות. לכל מסלול: פרופיל סיכון (1–7) והנחת תשואה ברירת מחדל.

## 4. מנוע החישוב

### 4.1 הנחות גלובליות (ניתנות לעריכה בכל סימולציה)
| פרמטר | ברירת מחדל | טווח |
|---|---|---|
| תשואה שנתית ריאלית — מסלול תלוי גיל | 3.74% (הנחת התקנון) | 0–8% |
| תשואה — מסלול מנייתי | 5.5% | |
| אינפלציה שנתית | 2% | 0–5% |
| עליית שכר ריאלית שנתית | 1.5% | 0–5% |
| הצגת ערכים | ריאלי (ברירת מחדל) / נומינלי | |
| גיל פרישה מתוכנן | גיל פרישה חוקי | 60–75 |

**כלל תצוגה:** לכל תוצאה — הצגת 3 תרחישים: פסימי / מרכזי / אופטימי (± סטיית הנחות), לא מספר יחיד.

### 4.2 הקרנת צבירה (לכל מוצר, חודשי)
```
יתרה(t+1) = יתרה(t) × (1 + r_חודשי)
          + הפקדה(t) × (1 − דמי_ניהול_מהפקדה)
          − עלות_כיסויים(t)
          − יתרה(t) × (דמי_ניהול_מצבירה / 12)
```
- הפקדה צמודה לתוואי השכר; עלויות כיסויים עולות עם הגיל (טבלת תעריפים).
- בקרן מקיפה: פיצול תשואה — 30% הבטחת תשואה (5.15% מובטח — פרמטר) + 70% תשואת שוק.

### 4.3 חישוב קצבה צפויה
```
קצבה_חודשית = צבירה_בפרישה / מקדם_המרה
```
- **מקדם המרה:** לפי טבלאות תקנון (תלוי שנת לידה, גיל פרישה, מגדר בפוליסות ישנות, מסלול קצבה: הבטחת 60/120/180/240 תשלומים, % קצבת שאיר לבן זוג 0–100%).
- בביטוח מנהלים עם מקדם מובטח — שימוש במקדם הפוליסה; הצגת השוואה מול מקדם קרן פנסיה עדכני.
- **שיעור תחלופה:** קצבה כוללת (כל המקורות + אזרח ותיק) ÷ שכר אחרון נטו. יעד מומלץ: 70%+ ; התראה מתחת ל-60%.

### 4.4 מנוע עלויות (דמי ניהול)
- עלות שנתית בש"ח לכל מוצר + עלות מצטברת עד פרישה.
- "מחיר הפער": הפרש הצבירה בפרישה בין דמי הניהול בפועל לבין ממוצע השוק ולמינימום המושג בשוק.
- תקרות חוקיות (פרמטרים): פנסיה 6%/0.5%, גמל וביטוח 4%/1.05%, השתלמות 2% מצבירה, ברירת מחדל נבחרות.
- דגל התראה: דמי ניהול מעל אחוזון 75 בשוק.

## 5. מודול תרחישים — "מה קורה אם" (ליבת המערכת)

לכל תרחיש: המערכת מריצה את המצב בכל גיל רלוונטי (או בגיל שנבחר), לפי הסטטוס המשפחתי, ומציגה **חלוקה מסודרת: מי מקבל, כמה, ממי, לכמה זמן, ומה המס**.

### 5.1 תרחיש מוות — לפני פרישה
לכל מוצר בנפרד + סיכום כולל:

| מוצר | מה קורה |
|---|---|
| **קרן פנסיה** | קצבת שארים חודשית: אלמן/ה 60% מהשכר הקובע (לכל החיים), יתום 40% (עד גיל 21), סה"כ עד 100%. אין בן זוג ואין יתומים → מוטבים/יורשים מקבלים את הצבירה כסכום חד-פעמי |
| **ביטוח מנהלים** | סכום ביטוח למקרה מוות + צבירה — למוטבים (חד-פעמי או קצבה) |
| **קופת גמל / השתלמות** | הצבירה למוטבים; פטור ממס רווח הון במשיכה עד תקרה/תנאים |
| **ותיקה / תקציבית** | קצבת שארים לפי תקנון (בד"כ 40–60% לאלמן/ה) |

**מטריצת תרחישים אוטומטית:** המערכת מחשבת מוות בגילאים 30/40/50/60/ערב-פרישה × סטטוס (נשוי+ילדים קטנים / נשוי בלי ילדים / רווק / גרוש עם ילדים) ומציגה:
- סך הכנסה חודשית למשפחה שתיוותר, מול צורכי המחיה (יעד: 70–80% מההכנסה כיום).
- **פער כיסוי** בש"ח — והמלצה (הגדלת שארים / ריסק משלים / ביטול כפילויות).
- התראות: ויתור שארים שפג תוקפו, מוטבים לא מעודכנים אחרי גירושין/נישואין, ילד שעבר גיל 21.

### 5.2 תרחיש מוות — אחרי פרישה
לפי מסלול הקצבה שנבחר: קצבת שאיר לבן זוג (% שנבחר), יתרת תשלומים מובטחים ליורשים, השוואת מסלולי קצבה (מקסימלית ללא שאירים מול 100% שאירים) עם נקודות איזון.

### 5.3 אובדן כושר עבודה (נכות)
- קצבת נכות מהקרן: עד 75% מהשכר הקובע, מלאה/חלקית, + שחרור מהפקדות (הקרן ממשיכה להפקיד).
- אכ"ע בביטוח מנהלים: לפי הגדרה עיסוקית, פרנצ'יזה, תיאום מול ביטוח לאומי.
- זיהוי **חוסר ביטוח**: שכר לא מבוטח (בונוסים, שעות נוספות), עצמאי ללא אכ"ע.
- תקופת אכשרה פעילה → אזהרה על החרגת מצב קודם.

### 5.4 עזיבת עבודה / הפסקת הפקדות
- השלכת משיכת פיצויים: פגיעה בקצבה (עד ~35–40% מהקצבה מגיעה מרכיב הפיצויים) + אובדן פטור עתידי; השוואה: משיכה מול רצף קצבה מול רצף זכויות.
- **ריסק זמני (הסדר ביטוח):** שמירת כיסויים עד 24 חודשים — עלות והתראת פקיעה (5 חודשים אוטומטיים).
- הקפאה: שחיקת צבירה מדמי ניהול ללא הפקדות; איבוד ותק ביטוחי אם עברו >5 חודשים.

### 5.5 תרחישי החלטה נוספים
| תרחיש | פלט |
|---|---|
| פרישה מוקדמת / דחויה (60–75) | קצבה לכל גיל פרישה, מקדם משתנה, גרף רגישות |
| שינוי מסלול השקעה | הפרש צבירה צפוי + הפרש סיכון (תרחישי שוק) |
| ניוד בין גופים | הפרש דמי ניהול מצטבר; אזהרות: אובדן מקדם מובטח, אכשרה מחדש, אובדן ותק |
| הגדלת הפקדה / הפקדה עצמאית | תוספת קצבה לכל 100 ₪ הפקדה + הטבת המס |
| היוון קצבה בפרישה | סכום חד-פעמי מול הפחתת קצבה, השלכת מס |

## 6. מודול מיסוי (פרמטרים לפי שנת מס)

### 6.1 בהפקדה
- **שכיר:** זיכוי 35% (סעיף 45א) על הפקדות עובד עד תקרת הכנסה מזכה; ניכוי (סעיף 47); זקיפת שווי על הפקדות מעסיק מעל התקרה (7.5% מ-3× שכר ממוצע).
- **עצמאי:** ניכוי עד 11% + זיכוי עד 5.5% מההכנסה המזכה; חובת הפקדה לעצמאי (מדרגות 4.45%/12.55%) + קנס אי-הפקדה.
- קרן השתלמות: תקרות הטבה שכיר/עצמאי; מעל התקרה — מס רווח הון על הרווחים.
- מחשבון "כמה מס חסכת השנה" + "כמה נשאר לנצל עד סוף השנה".

### 6.2 בפרישה
- **פטור על קצבה מזכה (סעיף 9א):** 52% מתקרת הקצבה המזכה (2025 — עולה מדרגתית ל-67% ב-2028+; פרמטר לפי שנה).
- **קיבוע זכויות (161ד):** סימולטור מלא — ניצול הפטור בין קצבה פטורה, היוון פטור ופיצויים פטורים; "נוסחת הקיזוז" על פיצויים שנמשכו ב-32 השנים שקדמו.
- פיצויים: פטור עד תקרה × שנות ותק; פריסת מס עד 6 שנים.
- תיקון 190: הפקדה לגמל למשיכה כקצבה מוכרת / משיכה הונית ב-15% נומינלי (בתנאים).
- מס על קצבה חייבת לפי מדרגות + תיאום מס בין מקורות.

## 10א. מודול בינה מלאכותית (AI)

המערכת תשולב עם מודל שפה (LLM) לצורך יכולות הסבר, ניתוח ושיחה. התמיכה תהיה **רב-ספקית**: Claude (Anthropic) ו-ChatGPT (OpenAI), עם אפשרות להרחבה עתידית.

### 10א.1 מסך הגדרות AI (Settings)

| שדה | סוג | הערות |
|---|---|---|
| ספק AI | enum | `anthropic` (Claude) / `openai` (ChatGPT) / כבוי |
| מפתח API | secret | נשמר מוצפן (at-rest); מוסתר בתצוגה; כפתור "בדוק חיבור" |
| מודל | בחירה | Anthropic: `claude-opus-4-8` (ברירת מחדל מומלצת), `claude-sonnet-5` (עלות/ביצועים), `claude-haiku-4-5` (מהיר וזול). OpenAI: רשימת מודלים עדכנית מה-API |
| רמת מאמץ / Reasoning | enum | ל-Claude: `effort` (low/medium/high) — high לניתוחים, low לצ'אט מהיר |
| תקציב חודשי | מספר | תקרת הוצאה ב-$ + מונה שימוש (טוקנים ועלות) והתראה ב-80% |
| מקסימום טוקנים לתשובה | מספר | ברירת מחדל 16,000; streaming לתשובות ארוכות |
| אנונימיזציה של PII | toggle | ברירת מחדל: **פעיל** (ראה 10א.4) |
| שמירת היסטוריית שיחות | toggle | מקומית בלבד |

### 10א.2 יכולות AI במערכת

| יכולת | תיאור | קלט שנשלח למודל |
|---|---|---|
| **הסבר בשפה חופשית** | כפתור "הסבר לי" ליד כל תוצאה — המודל מקבל את "עקבות החישוב" (סעיף 12) ומסביר בעברית פשוטה | נתוני חישוב אנונימיים |
| **יועץ צ'אט** | שאלות חופשיות על התיק ("מה יקרה לקצבה אם אפרוש ב-62?") — המודל מקבל את תמונת המצב ומפעיל את מנוע החישוב דרך **Tool Use** (המודל קורא לפונקציות המערכת, לא מחשב בעצמו!) | תמצית תיק אנונימית + הגדרות כלים |
| **קליטת דוח שנתי** | העלאת PDF של דוח שנתי מגוף מוסדי → חילוץ מובנה של יתרות, דמי ניהול ומסלולים (Structured Outputs עם סכמת JSON) → אישור ידני של המשתמש לפני שמירה | קובץ ה-PDF (בהסכמה מפורשת) |
| **ניסוח דוחות** | הפיכת ממצאי מנוע התובנות (7.3) לטקסט דוח קריא ומותאם ללקוח | ממצאים אנונימיים |
| **סיכום שינויים** | "מה השתנה בתיק שלי מאז הרבעון הקודם" בשפה טבעית | דלתא של נתונים |

**עיקרון ברזל:** ה-AI לעולם אינו מחשב מספרים פנסיוניים בעצמו. כל חישוב מתבצע במנוע הדטרמיניסטי של המערכת; המודל רק מסביר, מנסח, מחלץ נתונים ומפעיל כלים. כל תשובת AI מסומנת בממשק כ"נוצר על ידי AI" עם דיסקליימר.

### 10א.3 ארכיטקטורה

- **שכבת הפשטה (Provider Abstraction):** ממשק אחיד `IAiProvider` עם מימושים `AnthropicProvider` ו-`OpenAiProvider` — החלפת ספק ללא שינוי בלוגיקה העסקית.
- **SDK רשמיים בלבד:** Anthropic SDK (`anthropic` / `@anthropic-ai/sdk`) מול Messages API; OpenAI SDK הרשמי. לא לממש HTTP ידני.
- **Tool Use:** הגדרת כלים כגון `calc_projection`, `calc_death_scenario`, `get_portfolio_summary`, `calc_tax` — המודל קורא להם והמערכת מחזירה תוצאות מהמנוע.
- **Structured Outputs:** לחילוץ נתונים מדוחות — סכמת JSON קשיחה עם ולידציה (ב-Claude: `output_config.format`).
- **Streaming:** תשובות צ'אט והסברים מוזרמים לממשק בזמן אמת.
- **Prompt Caching:** ה-system prompt (הסבר על מבנה הנתונים והמונחים הפנסיוניים) קבוע וניתן לקישוי (cache) להוזלת עלויות בשיחות מרובות.
- **עמידות:** טיפול בשגיאות מסווג (rate limit / אימות / רשת), retry עם backoff (מובנה ב-SDK), ומצב degraded — המערכת מתפקדת במלואה גם כשה-AI כבוי או לא זמין.

### 10א.4 פרטיות ואבטחה ב-AI

- **אנונימיזציה לפני שליחה (ברירת מחדל):** הסרת שם, ת"ז, מספרי חשבון/פוליסה; שליחת נתונים מספריים והקשר בלבד. מיפוי מזהים (`מוצר-1`, `מוצר-2`) נשמר מקומית.
- **הסכמה מפורשת:** דיאלוג הסכמה בהפעלה ראשונה + חיווי קבוע כשמידע נשלח לספק חיצוני; שליחת PDF מלא (דוח שנתי) דורשת אישור פר-קובץ.
- **מפתח בבעלות המשתמש/הארגון:** המערכת אינה מתווכת מפתחות; כל לקוח עם המפתח שלו (או מפתח ארגוני ליועצים).
- **יומן AI:** תיעוד כל קריאה (מועד, יכולת, כמות טוקנים, עלות) — ללא שמירת תוכן רגיש ביומן.
- ציון בסעיף הרגולציה: תוכן AI אינו ייעוץ פנסיוני; באחריות בעל הרישיון לאמת לפני מסירה ללקוח.
